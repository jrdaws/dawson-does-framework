STEP 2: USAGE PATTERNS
----------------------

### Navigation Icons (inherit text color)

```tsx
// These use currentColor - they inherit from parent text color
<button className="text-gray-500 hover:text-indigo-500 transition-colors">
  <img 
    src="/images/configurator/nav/home.svg" 
    alt="Home"
    className="w-6 h-6"
  />
</button>

// Or with inline SVG for better currentColor support:
import HomeSvg from '@/public/images/configurator/nav/home.svg';

<HomeSvg className="w-6 h-6" />
```

### State Badges (self-colored)

```tsx
// Badges have semantic colors baked in
<div className="relative">
  <NavIcon />
  <span className="absolute -top-1 -right-1">
    <img 
      src="/images/configurator/badges/complete.svg" 
      alt="Complete"
      className="w-4 h-4"
    />
  </span>
</div>
```

### Category Icons (active/inactive states via CSS)

```tsx
// Active state (default)
<img 
  src="/images/configurator/categories/auth.svg" 
  alt="Authentication"
  className="w-8 h-8"
/>

// Inactive state
<img 
  src="/images/configurator/categories/auth.svg" 
  alt="Authentication"
  className="w-8 h-8 opacity-40"
/>

// Hover state
<img 
  src="/images/configurator/categories/auth.svg" 
  alt="Authentication"
  className="w-8 h-8 opacity-40 hover:opacity-70 transition-opacity"
/>
```

### Step Indicators

```tsx
// Choose the right SVG based on step state
const getStepIcon = (state: 'active' | 'complete' | 'pending' | 'locked' | 'optional' | 'error') => {
  return `/images/configurator/steps/${state}.svg`;
};

<img src={getStepIcon('active')} alt="Current step" className="w-6 h-6" />
```
