# Task: Add Integrations to Flagship-SaaS Template

**Priority:** P3 (Low)
**Assigned To:** Template Agent
**Created:** 2025-12-23 06:30
**Quality Score:** 7.8/10

---

## Context

The Template Agent's quality audit noted that `flagship-saas` has no bundled integrations, making it hard to differentiate from the basic `saas` template. Per the audit recommendation, add integrations to make it a more complete "batteries-included" starter.

## Vision Alignment

- **Export-First**: Flagship includes everything for quick deployment
- **Zero Lock-In**: Integrations are optional enhancements
- **Developer-Centric**: Save developers setup time

## Task Objectives

### 1. Copy Core Integrations from saas

Copy these integration directories from `templates/saas/integrations/` to `templates/flagship-saas/integrations/`:

**Required (copy as-is):**
- `auth/supabase/` - Authentication
- `db/supabase/` - Database
- `payments/stripe/` - Payments

**Premium (copy as-is):**
- `analytics/posthog/` - Product analytics
- `email/resend/` - Transactional emails

### 2. Add Storage Integration (if UploadThing completed)

If Integration Agent completed UploadThing integration:
- Copy `storage/uploadthing/` from saas template
- Otherwise, note this as a follow-up task

### 3. Update template.json

Update `templates/flagship-saas/template.json`:

```json
{
  "name": "flagship-saas",
  "displayName": "Flagship SaaS",
  "description": "Complete SaaS starter with auth, database, payments, analytics, and email",
  "version": "2.0.0",
  "category": "saas",
  "features": [
    "Authentication (Supabase)",
    "Database (Supabase)",
    "Payments (Stripe)",
    "Analytics (PostHog)",
    "Email (Resend)"
  ],
  "integrations": {
    "auth": ["supabase"],
    "db": ["supabase"],
    "payments": ["stripe"],
    "analytics": ["posthog"],
    "email": ["resend"]
  }
}
```

### 4. Update README.md

Update `templates/flagship-saas/README.md` to document:
- All included integrations
- Required environment variables
- Getting started with each integration
- Differentiation from basic saas template

### 5. Verify Build

```bash
# Test export with integrations
framework export flagship-saas ./test-flagship --auth supabase --db supabase

# Verify build
cd test-flagship
npm install
npm run build

# Cleanup
cd ..
rm -rf test-flagship
```
